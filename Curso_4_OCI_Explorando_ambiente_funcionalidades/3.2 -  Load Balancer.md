# ⚖️ OCI Load Balancer

## 📌 O que é Load Balancer?

O **Load Balancer** é um serviço da OCI que distribui automaticamente o tráfego entre múltiplos servidores.

### Funções principais:

- ⚖️ Balancear requisições
- 🚀 Melhorar performance
- 🛡️ Alta disponibilidade
- 🔄 Redundância
- 🔐 Segurança avançada

---

## 🚨 Problemas sem Load Balancer

- 🖥️ Servidor sobrecarregado
- ❌ Queda de serviço
- 🐢 Lentidão
- 🚫 Falta de redundância
- 💸 Perda de vendas em picos de acesso

### Exemplo:

🛒 E-commerce em promoção de final de ano.

---

## 🧩 Tipos de Load Balancer

## ⚙️ Load Balancer Tradicional

Mais inteligente e configurável.

### Pode ser:

- 🌍 Público → recebe tráfego da internet
- 🔒 Privado → uso interno (ex: banco de dados)

### Modos:

- 📈 Flexível → mínimo e máximo de banda
- 🤖 Dinâmico → ajuste automático

---

## ⚡ Network Load Balancer

Opera nas camadas 3 e 4.

### Ideal para:

- 🚀 Alta velocidade
- 📶 Baixa latência
- 📈 Grande volume de tráfego

---

# 🛠️ Criação do Load Balancer

## Caminho:

```text
Networking > Load Balancer
Exemplo:
Nome: teste1233
Tipo: Público
Subnet: Pública
Banda: Min 10 / Max 10
```

# 🖥️ Backends
Servidores que receberão o tráfego.

Exemplo:
🖥️ Instância 1

🖥️ Instância 2

# 🌐 Porta do Serviço
Exemplo usando Apache Tomcat:
```
8080
8081
```

# ❤️ Health Check
Verifica se servidor está saudável.

Importante:
Se serviço usa porta 8081, o health check também deve usar 8081.

# 👂 Listener
Responsável por escutar conexões.

Exemplo:
HTTP porta 8081

HTTPS (requer certificado)

# 🔐 Security Lists (VCN)
Necessário liberar acesso.

Regra de Entrada:
```
Source: 0.0.0.0/0
Porta: 8080
```

➡️ Permite acesso externo.

# 🧪 Teste do Balanceamento
Ao acessar IP público do Load Balancer:
```
Refresh (F5)
```

🖥️ Server 1

🖥️ Server 2
