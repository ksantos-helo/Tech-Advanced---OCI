# ☁️ OCI Governance & Administration — Cloud Advisor

# 🎯 O que é o Cloud Advisor?

O **Cloud Advisor** é um serviço da **OCI (Oracle Cloud Infrastructure)** que:

- 💰 Otimiza custos
- ⚡ Melhora desempenho
- 🔐 Aumenta segurança
- 🌍 Melhora alta disponibilidade
- 📊 Gera recomendações automáticas

---

# 🧠 Objetivo do Cloud Advisor

O serviço analisa automaticamente os recursos da tenancy e identifica:

- Recursos ociosos
- Configurações inseguras
- Problemas de desempenho
- Falhas de disponibilidade

---

# 💰 Recomendações de Economia de Custos

## 📌 Recursos subutilizados

O Cloud Advisor identifica:

- 🖥️ Instâncias ociosas
- 💾 Volumes sem uso
- 🔌 Recursos desligados mas ainda cobrando

---

## 🖥️ Instâncias Ociosas

### 🚨 Problema
Instâncias com:
- Baixo uso de CPU
- Baixo uso de memória
- Instâncias desligadas

continuam gerando custos.

---

## ✅ Solução sugerida
- Encerrar recursos
- Redimensionar instâncias
- Remover máquinas desnecessárias

---

# 💾 Volumes Não Anexados

## 📌 Tipos identificados

- Block Volumes
- Boot Volumes

---

## 🚨 Problema
Volumes não utilizados continuam gerando cobrança.

---

## ✅ Recomendação
- Excluir volumes não anexados

---

# ⚡ Recomendações de Performance

## 📈 Ajustes automáticos

O Advisor sugere melhorias para:
- Banco de dados
- Volumes de bloco
- Recursos sobrecarregados

---

# 🔄 Auto-Tuning

## ✅ O que é?
Ajuste automático de desempenho.

---

## 📌 Exemplo
Ativar auto-tuning em:
- 💾 Block Volumes

---

## 🎯 Benefícios
- Melhor desempenho
- Ajuste automático conforme uso
- Otimização contínua

---

# 🌍 Alta Disponibilidade (High Availability)

## 🚨 Problema identificado

Instâncias concentradas em:
- Um único Fault Domain

---

## ✅ Recomendação
Distribuir recursos entre:
- Diferentes Fault Domains

---

## 🎯 Benefícios
- Maior resiliência
- Menor risco de indisponibilidade
- Melhor tolerância a falhas

---

# 🔐 Recomendações de Segurança

O Cloud Advisor identifica:

- 🔓 Recursos públicos desnecessários
- 🌐 IPs públicos mal configurados
- 🚪 Portas abertas indevidamente
- ⚠️ Configurações inseguras

---

# 🚨 Exemplos de Problemas

## ❌ Exposição pública indevida
- Recursos acessíveis sem necessidade

## ❌ Regras inseguras
- Security Lists abertas demais

## ❌ Atividades suspeitas
- Configurações consideradas críticas

---

| Categoria | Objetivo |
|---|---|

# 🛠️ Aplicando Recomendações

## ✅ Funcionalidade
Muitas ações podem ser executadas diretamente pelo painel.

---

## 📌 Exemplos
- Remover recursos
- Ajustar configurações
- Ativar auto-tuning

---

# 📋 Work Requests

## 🎯 O que são?

---

## ✅ Permitem acompanhar:
- Status da execução
- Histórico de alterações
- Solicitações realizadas

---

# 📌 Informações exibidas

- ✅ Concluído
- ❌ Falhou
