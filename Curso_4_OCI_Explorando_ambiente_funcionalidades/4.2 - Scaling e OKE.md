# ☁️ OCI Oracle Certification - Scaling & OKE (Resumo para Estudos)

---

# 📈 O que é Scaling?

**Scaling (Escalonamento)** é a capacidade de **aumentar ou reduzir recursos** conforme a demanda da aplicação.

---

## 🔝 Tipos de Scaling

### 🔼 Vertical Scaling (Scale Up/Down)

- Ajusta recursos da **mesma máquina**
- Exemplo:
  - ➕ CPU
  - ➕ Memória

⚠️ **Atenção para prova:**
- Pode causar **downtime ⛔**
- Boa prática: fazer fora do horário de produção 🕒

---

### 🔁 Horizontal Scaling (Auto Scaling)

- Adiciona/remove **novas instâncias automaticamente**
- Ideal para aplicações com variação de carga

📌 Exemplo:
- CPU > 70% → ➕ cria nova instância  
- CPU < 20% → ➖ remove instância  

---

# ❌ Problemas sem Scaling

- 📉 Recursos insuficientes em picos
- 💸 Desperdício de infraestrutura
- 🧩 Dificuldade com microsserviços
- 🐢 Alta latência / resposta lenta

---

# 🛒 Exemplo Real (E-commerce)

Durante picos de acesso:

## Problema:
- Lentidão 😵
- Quedas ❌

## Solução:
- 🔁 Auto Scaling
- ⚖️ Load Balancer
- 🚀 Instâncias sob demanda

---

# ⚙️ Como funciona o Auto Scaling na OCI

## 🧱 Etapas:

1. 📸 Criar **Image (template)** da instância  
2. 📦 Criar **Instance Pool** (grupo de instâncias)  
3. 📏 Definir **regras de scaling**  

---

# 🧠 O que é OKE?

**OKE (Oracle Kubernetes Engine)** é um serviço gerenciado de **Kubernetes** na OCI.

👉 Responsável por:
- Orquestrar containers
- Gerenciar escalabilidade
- Garantir alta disponibilidade

---

# 🆚 VM vs Container

## 🖥️ Máquina Virtual (VM)

- Possui:
  - SO completo
  - CPU, memória e disco dedicados
- Mais pesada 🐘

---

## 📦 Container (Docker)

- Compartilha o sistema operacional
- Leve e rápido ⚡
- Alta portabilidade 🌍

---

# 🐳 Docker vs ☸️ Kubernetes

- 🐳 **Docker** → gerencia containers  
- ☸️ **Kubernetes** → orquestra containers  

📌 Trabalham juntos!

---

# 🏗️ Arquitetura Kubernetes (OKE)

## 🔹 Node (Nó)

- Máquina onde os containers rodam
- Também chamado de **Worker Node**

---

## 📦 Pod

- Menor unidade do Kubernetes
- Contém **1 ou mais containers**

---

## 📚 Node Pool

- Conjunto de nós

---

## 🎛️ Control Plane

- Gerencia:
  - Nodes
  - Pods
  - Cluster

✅ Gerenciado pela OCI  
💰 Sem custo adicional

---

## 💾 Banco de Dados (etcd)

- Armazena:
  - Configurações
  - Estado do cluster
  - Backups

---

# ☁️ Tipos de Cluster no OKE

## 🏆 Cluster Aprimorado (Enhanced)

- 🔒 SLA (garantia contratual)
- 🚀 Recursos completos
- ☁️ Suporte a **Virtual Nodes**

---

## 🧩 Cluster Básico

- 📉 Sem SLA
- 📜 Possui SLO (apenas objetivo)
- ⚙️ Recursos limitados

---

# 🧱 Tipos de Nodes

## ☁️ Virtual Node (Serverless)

- Totalmente gerenciado pela OCI
- Sem precisar administrar infraestrutura
- Disponível apenas no cluster aprimorado

---

## ⚙️ Managed Node

- Você gerencia
- Mais controle
- Usado em cluster básico

---

# 🎯 O que Cai na Prova

✅ Vertical Scaling = aumentar recursos da VM  
✅ Horizontal Scaling = adicionar instâncias  
✅ Auto Scaling = automático baseado em métricas  
✅ Instance Pool = grupo de instâncias  
✅ Docker = container  
✅ Kubernetes = orquestração  
✅ Pod = menor unidade  
✅ Node = máquina do cluster  
✅ SLA = garantia contratual  
✅ SLO = apenas objetivo  

---

# 🧠 Dicas de Certificação

- 🔥 Precisa de **elasticidade automática** → Horizontal Scaling  
- ⚡ Precisa de ajuste rápido de CPU/RAM → Vertical Scaling  
- 📦 Aplicações modernas (microservices) → Containers + Kubernetes  
- ☁️ Quer menos gerenciamento → OKE + Virtual Nodes  

---

# 🏆 Resumo Final

O **Scaling** garante performance e economia, enquanto o **OKE** permite gerenciar aplicações modernas com containers de forma escalável, resiliente e automatizada. 🚀
