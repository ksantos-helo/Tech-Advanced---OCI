# ☁️ OCI Oracle Certification - Compute & Instances (Resumo para Estudos)

## 🖥️ O que é Compute na OCI?
O serviço Compute da OCI permite criar recursos computacionais na nuvem, como:
↪️ 🖥️ Virtual Machines (VMs) → máquinas virtuais

↪️ 🏢 Bare Metal Servers → servidores físicos dedicados


São usados para hospedar aplicações, bancos de dados, APIs, sistemas corporativos e muito mais.

## 🚀 Vantagens do Compute na OCI
🔄 Flexibilidade
Escolha o tipo ideal de servidor:

↪️ VM para cargas comuns

↪️ Bare Metal para alto desempenho

📈 Escalabilidade
Criar ou remover instâncias rapidamente conforme necessidade.

## 🔐 Segurança
Controle total com:

↪️🔥 Firewall

↪️🌐 VCN

↪️📁 Compartments

↪️🔒 Subnets públicas e privadas


⚙️ Customização
Escolha:

↪️ Sistema operacional (Linux, Windows, Ubuntu etc.)

↪️ CPU

↪️ Memória RAM

↪️ Armazenamento



##  ❌ Problemas resolvidos pela OCI Compute
💸 Alto custo de hardware local
Servidores físicos exigem:

↪️compra de equipamentos

↪️manutenção

↪️energia

↪️segurança física

↪️sala-cofre


🐢 Escalabilidade lenta

Comprar e instalar novos servidores demora semanas.

🧠 Baixo desempenho técnico

Limitações para IA, Big Data e workloads pesados.

😵 Complexidade de gerenciamento

Infraestrutura local exige mais equipes e tempo.

## 🛠️ Criando Instância na OCI
Caminho no Console:
```
☰ Menu > Compute > Instances > Create Instance
```

## 📌 Configurações Importantes
🏷️ Nome e Compartment
Escolha:

↪️nome da VM

↪️compartment destino



## 🌍 Placement
Define onde a máquina ficará:

↪️Availability Domain

↪️Fault Domain


✅ Recomendado para prova:
Let Oracle choose best fault domain


## 💿 Image and Shape
Image (Sistema Operacional)
Pode escolher:

↪️Oracle Linux

↪️Ubuntu

↪️Red Hat

↪️CentOS

↪️Windows


Shape (Hardware)
Define recursos:

↪️OCPU

↪️RAM

↪️Tipo da máquina


## 🖥️ Tipos de Shape
☁️ Virtual Machine

↪️compartilhada via hypervisor

↪️custo menor

↪️ideal para maioria dos casos


🏢 Bare Metal

↪️servidor físico exclusivo

↪️máquima performance

↪️workloads pesados



## 🌐 Rede da Instância
VCN
Escolha:


criar nova VCN


usar existente


VNIC
Placa de rede virtual da instância.
IP Primário
Pode atribuir:


IPv4 privado


IPv4 público


📌 Rede pública = IP público automático

## 💾 Armazenamento
Boot Volume
Disco principal do sistema operacional.
Block Volume
Disco adicional (HD/SSD secundário).
Termos importantes:


📎 Attach = anexar volume


❌ Detach = remover volume



## 🔑 SSH Keys
Para acesso seguro Linux.
Opções:


criar novo par de chaves


upload chave pública


colar chave manualmente


Comando exemplo:
```
ssh -i chave.key usuario@IP
```

## ☁️ Cloud Shell
Ferramenta nativa OCI com terminal Linux no navegador.
Vantagens:


não precisa instalar programas


pode usar ssh-keygen


conectar direto nas VMs



## 🔄 Live Migration
Sempre deixar habilitado ✅
Se houver falha física no host:


OCI move a instância automaticamente


sem interrupção



## 🎯 O que Cai na Prova

Memorize:
✅ Compute = serviço de servidores na nuvem

✅ VM = máquina virtual

✅ Bare Metal = servidor físico dedicado

✅ Shape = CPU + RAM

✅ Image = sistema operacional

✅ Boot Volume = disco principal

✅ Block Volume = disco extra

✅ VNIC = placa de rede virtual

✅ SSH = acesso remoto seguro

✅ Live Migration = migração sem downtime

## 🧠 Dica de Certificação
Se a pergunta falar em:


alto desempenho máximo → Bare Metal 🏢


baixo custo e flexibilidade → VM ☁️


disco adicional → Block Volume 💾


migração sem parada → Live Migration 🔄



## 🏆 Resumo Final
OCI Compute permite criar servidores rápidos, seguros, escaláveis e sob demanda, substituindo infraestrutura física cara e complexa.
