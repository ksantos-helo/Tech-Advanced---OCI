# ☁️ OCI Oracle Certification - Serverless & Oracle Functions (Resumo)

---

# ⚡ O que é Serverless?

**Serverless** = modelo onde você **não gerencia servidores**.

👉 A OCI cuida de:
- Infraestrutura 🏢
- Escalabilidade 📈
- Disponibilidade 🔄

Você foca apenas no **código** 💻

---

# 🧠 O que é Oracle Functions?

- 🔹 Serviço **serverless** da OCI  
- 🔹 Executa **funções sob demanda**  
- 🔹 Baseado no **Fn Project (open source)**  
- 🔹 Execução em **containers gerenciados**

---

# 🚀 Benefícios do Oracle Functions

- ❌ Sem gerenciamento de servidores  
- 📈 Escala automática  
- 💰 Pague apenas pelo uso (pay-per-use)  
- 🔗 Integração com serviços OCI  
- ⚡ Execução rápida e paralela  

---

# ❌ Problemas resolvidos

- 💸 Infraestrutura cara  
- 🧱 Complexidade de gerenciamento  
- 📉 Escalabilidade ineficiente  
- ⏳ Recursos ociosos sendo pagos  

---

# 🎯 Caso Real (Processamento de Vídeos)

## Problema:
- Tarefa executada 1x por dia
- Servidor ficaria ocioso 😴

## Solução:
- ⚡ Oracle Functions
- Executa somente quando necessário
- 💰 Custo apenas na execução

---

# ⚙️ Como funciona o Oracle Functions

## 🧱 Estrutura:

1. 📦 Código → empacotado em container  
2. 📤 Enviado para OCI Registry  
3. 🎯 Configuração de gatilhos (triggers)  
4. 🚀 Execução sob demanda  

---

# 🔔 Formas de execução

- 🌐 API (requisições HTTP)
- 💻 CLI
- 📡 Eventos OCI (event-driven)

📌 Exemplo:
Upload no Object Storage → dispara função automaticamente

---

# 🔄 Fluxo de execução

1. Evento acontece ⚡  
2. Função é acionada 🔔  
3. Container é iniciado 📦  
4. Código executa 💻  
5. Resultado é retornado ✅  

---

# 🐳 Funções e Containers

- Funções rodam em **containers**
- Totalmente gerenciados pela OCI
- Baseados no **Fn Project (open source)**

---

# 💰 Modelo de cobrança

- ⏱️ Cobrado por execução  
- ❌ Sem uso = sem custo  

📌 Exemplo:
Executou às 14h → paga só por esse uso

---

# 🔗 Integrações comuns

- 📦 Object Storage  
- 🗄️ Banco de Dados  
- 🔔 Eventos OCI  
- 🌐 APIs externas  

---

# 🎯 O que cai na prova

✅ Serverless = sem servidor  
✅ Oracle Functions = FaaS (Function as a Service)  
✅ Execução sob demanda  
✅ Baseado em Fn Project  
✅ Usa containers  
✅ Event-driven (gatilhos)  
✅ Pay-per-use  

---

# 🧠 Dicas de Certificação

- 🔥 Tarefas event-driven → Oracle Functions  
- 💰 Reduzir custo com ociosidade → Serverless  
- ⚡ Execução rápida e automática → Functions  
- 🔗 Integrações com eventos → ideal para Functions  

---

# 🏆 Resumo Final

O **Oracle Functions** permite executar código sem gerenciar servidores, com escalabilidade automática, integração com eventos e cobrança apenas pelo uso. Ideal para aplicações modernas e orientadas a eventos. 🚀
