# 🔐 Autenticação e Autorização na OCI

## 📌 Conceitos Principais

## 🔑 Autenticação (AuthN)

Confirma **quem é o usuário**.

### Exemplos:

- 📧 E-mail + senha
- 👤 Usuário + senha
- 📲 MFA (autenticação multifator)

➡️ Responde: **Quem está acessando?**

---

## 🛡️ Autorização (AuthZ)

Define **o que o usuário pode fazer**.

### Exemplos:

- 👀 Apenas visualizar
- ✏️ Editar recursos
- 🗑️ Excluir recursos
- 👑 Administrar ambiente

➡️ Responde: **O que pode acessar?**

---

# ⚠️ Principais Problemas de Segurança

- 🚫 Acesso indevido a recursos sensíveis
- 🔓 Senhas fracas
- 🕵️ Roubo de credenciais
- 📄 Falta de auditoria
- 🔀 Dificuldade de gerenciar múltiplos sistemas

---

# ✅ Soluções na OCI

## 🔒 Política de Senhas

- 📏 Tamanho mínimo
- 🔠 Complexidade
- 🔄 Troca periódica

## 📲 MFA

Camada extra de segurança com:

- Google Authenticator
- Apps autenticadores

## 📋 Auditoria e Controle

Monitoramento de acessos e permissões.

---

# 🧩 OCI IAM Gerencia Tudo

O IAM controla:

- 👤 Usuários
- 👥 Grupos
- 📦 Compartments
- 📜 Políticas
- 🔐 Login e acesso

---

# 👤 Criação de Usuários

Ao criar usuário na OCI:

- Nome
- Sobrenome
- 📧 E-mail
- 👤 Username
- 👥 Grupos vinculados

### Exemplo de grupos:

- All Domain Users
- Administrators

---

# 👥 Grupos

Conjunto de usuários com permissões iguais.

### Exemplo:

📌 Grupo: `teste grupo`

Usuários adicionados herdam permissões do grupo.

---

# 📜 Políticas de Acesso

Definem o que grupos podem fazer.

### Exemplo:

Permitir grupo gerenciar armazenamento.

```text
manage file system
create volume group
```

Aplicado em:
📦 Compartimento específico

🌐 Domínio específico

# 🏢 Domínios

Usados para separar identidades e gerenciamento.

Exemplo:
domain-ex

# 🚀 Fluxo Completo

👤 Criar usuário

👥 Adicionar ao grupo

🏢 Criar domínio

📜 Criar política

📲 Ativar MFA

🔐 Usuário acessa conforme permissões

✔️ Ligadas a compartimentos

- “Grupo Dev pode gerenciar storage”

---

# 📦 Compartimentos
