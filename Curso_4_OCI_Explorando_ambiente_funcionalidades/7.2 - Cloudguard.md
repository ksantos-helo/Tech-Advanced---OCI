# 🧠 OCI Cloud Guard — Resumo

## 📌 O que é o Cloud Guard?
O **Cloud Guard** é um serviço de segurança da OCI que:

- 🔍 Monitora continuamente recursos  
- 🚨 Detecta configurações incorretas e atividades suspeitas  
- 🤖 Responde automaticamente a ameaças  
- 📊 Analisa riscos em tempo real  

---

## 🎯 Definindo os alvos (targets)

- O Cloud Guard trabalha com **targets (alvos)**
- Define **quais recursos serão monitorados**
- Exemplo:
  - Compartments
  - Redes
  - Instâncias

---

## 🔎 Detectando problemas

- Usa **detectors (detectores)** para identificar riscos:

Exemplos:
- 🌐 Instância pública indevida  
- 🪣 Bucket público  
- ⚠️ IP suspeito  

➡️ Ao detectar:
- Gera **alertas**
- Classifica por nível de risco (ex: crítico)

---

## ⚙️ Respondendo automaticamente

- Usa **responders (respondedores)**

Ações automáticas:
- 🛑 Parar instância exposta  
- 🔒 Tornar bucket privado  
- 🚫 Bloquear IP suspeito  

---

## 🔄 Fluxo de funcionamento

1. 🎯 Define target (alvo)  
2. 🔍 Detecta problemas (detectors)  
3. 🚨 Gera alertas  
4. 🤖 Executa ações automáticas (responders)  

---

## 🧪 Exemplo prático

- Problema: bucket público  
- Cloud Guard:
  1. Detecta  
  2. Classifica como crítico  
  3. Envia alerta  
  4. Corrige automaticamente (torna privado)  

---

# 🧩 Estruturando o monitoramento com Cloud Guard

## 🎯 Definindo os alvos a serem monitorados
- Escolha de compartments e recursos

## 🔍 Analisando configurações e atividades
- Detectores identificam riscos e falhas

## 🚨 Gerando alertas de segurança
- Problemas são classificados por criticidade

## 🤖 Implementando respostas automáticas
- Responders corrigem problemas sem intervenção
