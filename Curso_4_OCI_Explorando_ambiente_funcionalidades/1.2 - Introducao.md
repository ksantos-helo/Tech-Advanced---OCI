# ☁️ Introdução ao Oracle Cloud Infrastructure (OCI)

## 📌 Problemas do Ambiente On-Premises

Infraestrutura local (on-premises) apresenta desafios como:

- 💸 Alto custo de equipamentos
- 📉 Baixa flexibilidade para expansão
- 🖥️ Troca total de hardware para upgrade
- 🐢 Alta latência dependendo da rede/provedor
- 📄 Faturas imprevisíveis
- 📦 Limitações de espaço, memória e processamento

---

## 🚀 Vantagens do OCI

Com a Oracle Cloud Infrastructure temos:

- 💰 Previsibilidade de custos
- ⚡ Baixa latência
- 🔄 Escalabilidade de recursos
- 🔥 Ajuste "a quente" (aumentar recursos sem parar ambiente)
- ⚖️ Load Balancer
- 🛡️ Alta disponibilidade

# 🏢 Arquitetura OCI

## 📍 Região

Área geográfica com um ou mais datacenters.

## 🏬 Domínio de Disponibilidade (Availability Domain)

Datacenters independentes dentro da região.

### Benefícios:

- 🔌 Energia separada
- ❄️ Resfriamento independente
- 🌐 Rede isolada
- 🛡️ Tolerância a falhas

---

## ⚙️ Domínio de Falha (Fault Domain)

Divisão interna dentro do domínio de disponibilidade.

Cada domínio possui:

- ⚡ Energia independente
- 🖥️ Hardware separado
- 🔧 Proteção contra manutenção e falhas físicas

📌 Cada Availability Domain possui **3 Fault Domains**.

### Exemplo:

- 🖥️ Servidor principal em um Fault Domain
- 🔄 Servidor secundário em outro

Se um falhar, o outro assume.

---

# 🌐 Oracle Multicloud

Permite integrar OCI com outras nuvens:

- ☁️ AWS
- ☁️ Azure
- ☁️ Google Cloud

### Benefícios:

- 🔗 Conexão segura entre nuvens
- 🧩 Uso dos melhores serviços de cada provedor
- 🗄️ OCI para banco de dados, MySQL e HeatWave
- 🚀 Estratégia multinuvem eficiente

# 🧠 Resumo Final

OCI oferece uma infraestrutura moderna, escalável e segura, substituindo limitações do ambiente on-premises com alta disponibilidade, flexibilidade e integração multicloud.


# 🎯 O que cai na prova

✅ On-Premises = alto custo, baixa escalabilidade e manutenção complexa  
✅ OCI = escalabilidade elástica (aumenta/reduz recursos sob demanda)  
✅ OCI = modelo de custo previsível (pay-as-you-go)  
✅ OCI = alta disponibilidade e tolerância a falhas  
✅ OCI = balanceador de carga (Load Balancer)  
✅ Região = área geográfica com datacenters  
✅ Região pode ter 1 ou mais Availability Domains (ADs)  
✅ Availability Domain = datacenter isolado dentro da região  
✅ AD = energia, rede e refrigeração independentes  
✅ Fault Domain = divisão interna do AD para evitar falhas  
✅ Cada AD possui 3 Fault Domains  
✅ Fault Domain = anti-afinidade (evita que VMs fiquem no mesmo hardware)  

---

# 🌐 Oracle Multicloud

✅ Integra OCI com AWS, Azure e Google Cloud  
✅ Permite uso híbrido de serviços entre nuvens  
✅ OCI pode ser usada para banco de dados e analytics (ex: MySQL HeatWave)  
✅ Comunicação segura entre provedores de cloud
