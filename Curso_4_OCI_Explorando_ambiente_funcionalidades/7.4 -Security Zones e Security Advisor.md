# 🛡️ OCI Security Zones & Security Advisor — Resumo

## 📌 O que são Security Zones?
As **Security Zones** são áreas (compartments) com **regras de segurança obrigatórias**, que **não podem ser desativadas**.

- 🔒 Impõem políticas rígidas automaticamente  
- 🚫 Bloqueiam qualquer ação que viole essas regras  
- 🔐 Garantem segurança máxima por padrão  

### 🧠 Exemplo
- Bucket público → ❌ Negado automaticamente  
- Recurso sem criptografia → ❌ Não permitido  

📌 Analogia:  
👉 Como um **cofre à prova de falhas** — nada fora da política entra.

---

## ⚙️ Como funcionam

- Aplicadas a **compartments**
- Baseadas em **recipes (políticas predefinidas)**
- Integram com vários serviços:
  - 🌐 Rede (VCN)
  - 💾 Storage
  - 🖥️ Compute
  - 🗄️ Banco de dados

---

## 🚫 Comportamento

- Detecta violação de política  
- ❌ **Bloqueia automaticamente a operação**  
- Segurança **não pode ser desativada** (sem remover a zona)

---

## 🧠 O que é o Security Advisor?

O **Security Advisor** é um serviço que:

- 📊 Centraliza informações de segurança  
- 🔗 Integra:
  - Cloud Guard  
  - Security Zones  
  - Outros serviços  

- 💡 Fornece recomendações de boas práticas

---

## 🔍 Funções do Security Advisor

- 📈 Avalia postura de segurança  
- ⚠️ Identifica riscos  
- 🧭 Orienta configurações seguras  
- 🔐 Garante conformidade (ex: criptografia, acesso)

---

## 🧩 Exemplo prático

- Problema: bucket público  
- Security Advisor:
  - Detecta risco  
  - Recomenda ajuste  
  - Sugere uso de Security Zone  

---

# 🧠 Estruturando a segurança com Security Zones

## 🔒 Definindo áreas seguras obrigatórias
- Compartments com políticas rígidas

## 🚫 Bloqueando ações fora da política
- Qualquer violação é negada automaticamente

## ⚙️ Aplicando regras em múltiplos serviços
- Rede, compute, storage e banco de dados

---

# 🧠 Analisando segurança com Security Advisor

## 📊 Consolidando informações de segurança
- Visão centralizada

## 🔍 Identificando vulnerabilidades
- Avaliação contínua

## 🧭 Implementando boas práticas
- Recomendações automáticas
