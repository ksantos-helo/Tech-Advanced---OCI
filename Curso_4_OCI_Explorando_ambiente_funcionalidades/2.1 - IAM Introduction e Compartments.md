# 🔐 Oracle IAM na OCI

## 📌 O que é Oracle IAM?

**IAM (Identity and Access Management)** é o serviço da Oracle para:

- 👤 Gerenciar identidades
- 🔑 Controlar acessos
- 🛡️ Definir permissões
- 📋 Aplicar políticas de segurança
- 🔐 Melhorar autenticação

---

# ⚠️ Problemas Sem IAM

Sem controle de acesso, podem ocorrer:

- 🚫 Acesso sem controle
- 🕵️ Risco de violação
- 🔑 Gestão complexa de credenciais
- ❌ Falhas de autenticação
- 📄 Dificuldade em auditoria
- ⚖️ Problemas com LGPD / GDPR

---

# ✅ O que o IAM Resolve

Com Oracle IAM é possível:

- 👥 Criar usuários
- 👨‍👩‍👧‍👦 Criar grupos
- 📜 Definir políticas
- 🔒 Restringir acessos
- ✅ Liberar acessos específicos
- 📲 Usar MFA (autenticação multifator)

---

# 🧩 Componentes Principais do IAM

## 👤 Usuários

Contas individuais de acesso.

## 👥 Grupos

Conjunto de usuários com permissões em comum.

## 📜 Políticas

Regras que permitem ou negam acesso.

## 📦 Compartments

Estrutura lógica para organizar recursos.

## 🏢 Domínio / Tenancy

Conta principal da OCI.

## 🔑 Chaves e Tokens

Usados para:

- 🤖 Scripts
- ⚙️ Automação
- 🔗 APIs

---

# 📦 Compartments (Compartimentos)

## 📌 O que são?

Pastas lógicas usadas para:

- 📂 Organizar recursos
- 🔒 Isolar ambientes
- 👥 Controlar acessos
- 🏗️ Separar projetos

---

# 🧠 Exemplo Prático

## Empresa A e Empresa B

Criar:

- 📦 Compartment Empresa A
- 📦 Compartment Empresa B

Depois definir:

- 👥 Grupo A acessa Empresa A
- 👥 Grupo B acessa Empresa B

---

# 🌳 Hierarquia

É possível criar compartments dentro de compartments.

### Exemplo:

- Root
  - 📦 Curso OCI
    - 📦 Rede
    - 📦 Banco
    - 📦 Produção

---

# 🚀 Benefícios dos Compartments

- 📁 Organização
- 🔐 Segurança
- 👀 Controle de acesso
- 📊 Gestão eficiente
- 🏢 Separação por times/projetos

---

# 🧠 Resumo Final

✔️ Oracle IAM controla identidade e acesso na OCI.

✔️ Principais recursos:

- 👤 Usuários
- 👥 Grupos
- 📜 Políticas
- 📦 Compartments

✔️ Compartments funcionam como pastas seguras para organizar recursos.
