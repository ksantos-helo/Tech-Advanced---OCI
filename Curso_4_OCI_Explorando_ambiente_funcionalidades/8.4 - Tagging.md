# 🏷️ OCI Governance & Administration — Tags na OCI

# 🎯 O que são Tags?

As **Tags** na **OCI (Oracle Cloud Infrastructure)** são:

- 🏷️ Pares chave-valor
- 📦 Utilizadas para organizar recursos
- 🔎 Facilitam localização e gerenciamento

---

# 📌 Estrutura de uma Tag

Uma tag é composta por:

```text
chave = valor
```

---

# 🧾 Exemplos de Tags

## 🌍 Ambiente

```text
environment = production
```

---

## 🚀 Projeto

```text
project = alpha
```

---

# 🚨 Problemas Sem Uso de Tags

Sem tags, ambientes grandes podem apresentar:

- 🔍 Dificuldade para localizar recursos
- 📊 Falta de controle de custos
- 🧩 Ambientes desorganizados
- 🔐 Controle de acesso limitado

---

# ✅ Benefícios das Tags

## 📦 Organização

Permitem identificar recursos por:
- Projeto
- Ambiente
- Equipe
- Departamento

---

## 🔎 Facilidade de Busca

Recursos etiquetados ficam:
- Mais rápidos de localizar
- Mais simples de administrar

---

## 💰 Controle de Custos

Permitem:
- Monitorar custos por projeto
- Separar gastos por equipe
- Gerar relatórios organizados

---

## 🔐 Segurança e Controle de Acesso

É possível:
- Aplicar políticas baseadas em tags
- Restringir acesso conforme equipe
- Organizar permissões dentro do mesmo compartment

---

# 🏷️ Tipos de Tags na OCI

A OCI possui:

1️⃣ Free-form Tags (Tags Livres)  
2️⃣ Defined Tags (Tags Definidas)

---

# 1️⃣ Free-form Tags (Tags Livres)

## 🎯 Características

- Simples
- Flexíveis
- Sem validação
- Não exigem configuração prévia

---

# 📌 Estrutura

```text
environment = production
```

---

# ⚠️ Limitações

- ❌ Sem controle de valores
- ❌ Sem padronização
- ❌ Menor governança

---

# 2️⃣ Defined Tags (Tags Definidas)

## 🎯 Características

Mais avançadas e estruturadas.

---

# 🧩 Estrutura Completa

Uma Defined Tag possui:

- Namespace
- Chave
- Valor

---

# 📌 Exemplo

```text
operations.environment = production
```

---

# 🧠 Estrutura Detalhada

| Elemento | Valor |
|---|---|
| Namespace | operations |
| Chave | environment |
| Valor | production |

---

# ✅ Benefícios das Defined Tags

## 🔐 Maior Controle

Permitem:
- Validar valores permitidos
- Padronizar nomenclatura
- Aplicar governança

---

## 📋 Políticas de Acesso

Podem ser usadas em:
- IAM Policies
- Controle de recursos
- Segurança

---

## 📊 Melhor Auditoria

Facilitam:
- Rastreamento
- Compliance
- Organização corporativa

---

# 🗂️ Namespace nas Defined Tags

## 🎯 O que é Namespace?

É um agrupador lógico de tags.

---

# 📌 Exemplo

```text
operations.environment = production
```

Onde:
- operations → namespace
- environment → chave
- production → valor

---

# ⚠️ Importante Sobre Namespace

## ❌ Não pode ser excluído

Após criado:
- Não pode ser deletado

---

## ✅ Pode ser aposentado (retired)

Isso mantém:
- Histórico
- Auditoria
- Rastreamento das tags antigas

---

# 📚 Exemplo de Namespace Aposentado

Mesmo aposentando:

```text
project
```

O histórico das tags continua disponível.

---

# 🎯 Objetivo das Tags

As tags ajudam a:

- 📦 Organizar recursos
- 🔎 Localizar rapidamente serviços
- 💰 Controlar custos
- 🔐 Melhorar segurança
- 🧩 Separar ambientes

---

# 🧠 Resumo para Certificação OCI

## 🎯 Pontos Mais Importantes

✅ Tags são pares chave-valor  
✅ Facilitam organização de recursos  
✅ Melhoram gerenciamento de custos  
✅ Free-form Tags são simples e sem validação  
✅ Defined Tags possuem namespace  
✅ Defined Tags oferecem maior controle  
✅ Namespace não pode ser excluído  
✅ Namespace pode ser aposentado  
✅ Tags podem ser usadas em políticas IAM  

---

# ❓ Possíveis Questões de Certificação

## ❓ Qual a diferença entre Free-form e Defined Tags?

✅ Free-form:
- Simples
- Sem validação

✅ Defined Tags:
- Estruturadas
- Possuem namespace
- Permitem controle e governança

---

## ❓ Qual estrutura uma Defined Tag utiliza?

✅ Namespace + Chave + Valor

---

## ❓ O namespace pode ser excluído?

❌ Não

---

## ❓ O que pode ser feito com um namespace?

✅ Aposentar (retire)

---

## ❓ Para que servem as tags?

✅ Organização  
✅ Segurança  
✅ Controle de custos  
✅ Governança  

---

# 🚀 Dicas para Certificação OCI

📌 A Oracle costuma cobrar:

- 🏷️ Diferença entre Free-form e Defined Tags
- 🗂️ Conceito de Namespace
- 🔐 Uso de tags em políticas IAM
- 💰 Controle de custos com tags
- 📊 Governança usando Defined Tags
- 📚 Namespace aposentado (retired)
