# 🏢 Tenancy na OCI

## 📌 O que é Tenancy?

A **Tenancy** é a conta principal (conta mestre) criada ao registrar-se na OCI.

Ela representa a estrutura central da organização dentro da Oracle Cloud.

### Inclui:

- 👤 Identidade e acesso (IAM)
- ☁️ Recursos em nuvem
- 💰 Faturamento
- 📊 Monitoramento
- 🛡️ Governança e segurança

➡️ Tudo na OCI fica dentro da Tenancy.

---

# 🌳 Estrutura da Tenancy

A Tenancy é o nível mais alto da hierarquia.

### Exemplo:

```text
Tenancy
 ├── Compartment A
 ├── Compartment B
 └── Compartment C
```

📦 Compartments são filhos da Tenancy.

# ⚠️ Problemas de Má Configuração
Se a Tenancy não for bem organizada:

📂 Falta de organização

🚫 Acesso sem restrição

👥 Dificuldade com múltiplas equipes

💸 Custos imprevisíveis

📈 Escalabilidade descontrolada

🔐 Riscos de segurança

# ✅ Boas Práticas
👑 Não usar conta principal no dia a dia
A conta inicial possui acesso total.

🚫 Evite usar essa conta operacionalmente.

# 👤 Criar OCI Admin
Criar usuário administrador separado:

OCI Admin

Mesmo admin, pode ter restrições específicas.

# 👥 Criar Grupos Administrativos
Exemplo:

OCI-admin-group

DBA-group

Network-group

Compute-group

➡️ Cada grupo com responsabilidades específicas.

# 📜 Políticas IAM
Definem permissões dos grupos.

Exemplo:
Administrador de banco

Administrador de rede (VCN)

Administrador de instâncias

# 📦 Compartments
Usados para isolar recursos e responsabilidades.

Exemplo:
```
Compartment Teste1
 ├── Instâncias
 ├── Banco de Dados
 └── Storage
```

➡️ Facilita controle por equipe.

# 📲 Segurança Extra
MFA (Autenticação Multifator)
Adicionar segunda camada de segurança:

📱 App autenticador

Código temporário

# 🎯 Benefícios
🔐 Mais segurança

📊 Melhor auditoria

⚖️ Conformidade

👥 Gestão por equipes

💰 Controle de custos

📂 Organização escalável

# 🧠 Resumo Final
✔️ Tenancy = Conta principal da OCI

✔️ Tudo fica dentro dela:

IAM

Recursos

Billing

Segurança

✔️ Melhores práticas:

👤 Criar OCI Admin

👥 Separar grupos

📜 Aplicar políticas

📦 Usar compartments

📲 Ativar MFA

# 🧠 O que cai na prova (resumo final)

✔️ Tenancy = conta principal da OCI (raiz)  
✔️ Tudo pertence à Tenancy (IAM, recursos, billing)  
✔️ Compartimentos organizam a estrutura  
✔️ Não usar root no dia a dia  
✔️ Criar OCI Admin separado  
✔️ Usar grupos + policies para controle  
✔️ MFA aumenta segurança
