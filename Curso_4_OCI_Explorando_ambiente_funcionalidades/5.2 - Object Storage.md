# ☁️ OCI Object Storage — Resumo

## 📦 O que é Object Storage?
O **Object Storage** da OCI é um serviço de armazenamento escalável e de alto desempenho baseado em **objetos**.

- Armazena qualquer tipo de dado como **objeto**
- Ideal para dados **não estruturados**:
  - Logs
  - Vídeos
  - Imagens
  - Arquivos em geral
- Cada objeto possui:
  - 📄 Conteúdo
  - 🏷️ Metadados (*dados sobre dados*)

---

## 🪣 Estrutura: Buckets e Namespace

### 🔹 Buckets
- Local onde os objetos são armazenados
- Nome único dentro da tenancy
- Estrutura **plana** (sem pastas reais, apenas prefixos)

### 🔹 Namespace
- Contêiner lógico de nível superior
- Nome **global único**
- Usado na construção da URL

---

## 🌐 Acesso aos objetos

- Feito via **endpoint (URL)**
- Exemplo de estrutura:
https://objectstorage.<região>.oraclecloud.com/n/<namespace>/b/<bucket>/o/<objeto>


### 📌 Componentes da URL:
- Região (ex: sa-saopaulo-1)
- Namespace
- Bucket
- Nome do objeto

---

## 🧊 Níveis de Armazenamento (Tiers)

| Tier                  | Uso principal |
|----------------------|--------------|
| Standard             | Acesso frequente |
| Infrequent Access    | Acesso ocasional |
| Archive              | Raramente acessado |
| Auto-Tiering         | Ajuste automático |

---

## ⚙️ Recursos Importantes

- 🔄 **Lifecycle Management**
  - Automatiza movimentação entre tiers

- 🕓 **Versionamento**
  - Mantém múltiplas versões do objeto

- 🔐 **Criptografia**
  - Ativa por padrão (não pode desativar)

---

## 🏗️ Criação de Bucket

Passos básicos:
1. Menu ☰ → Storage
2. Object Storage & Archive Storage
3. Create Bucket
4. Definir:
   - Nome
   - Tier (ex: Standard)
   - Auto-Tiering (opcional)
5. Criar

---

## 📤 Upload de Objetos

- Acessar bucket
- Clicar em **Upload**
- Selecionar arquivo
- Enviar

---

## 🔓 Visibilidade e Acesso

### 🔒 Privado (padrão)
- Apenas usuários autorizados

### 🌍 Público
- Acessível via URL (⚠️ não recomendado)

### 🔑 Pre-Authenticated Request (PAR)
- Gera URL temporária com acesso controlado
- Possui:
  - Token de autenticação
  - Data de expiração (não editável depois)

---

## 🧪 Observações importantes

- URL direta só funciona se:
  - Bucket for público **OU**
  - Usar URL com PAR
- Tornar bucket público **não é boa prática**

---

# 🎯 O QUE MAIS CAI NA CERTIFICAÇÃO OCI

## 🔥 Pontos-chave (memorizar!)

- 📦 Object Storage = armazenamento de **dados não estruturados**
- 🪣 **Bucket** = onde ficam os objetos
- 🌐 **Namespace** = identificador global único
- 🔗 Estrutura da URL:
/n/<namespace>/b/<bucket>/o/<objeto>

- 🧊 Diferença entre Tiers:
- Standard → frequente
- Infrequent → pouco acesso
- Archive → raríssimo acesso
- 🔄 **Lifecycle Policy** automatiza movimentação entre tiers
- 🔐 **Criptografia sempre ativa (by default)**
- 🔓 Tipos de acesso:
- Private (padrão)
- Public (evitar)
- PAR (muito cobrado!)
- ⚠️ **Pre-Authenticated Request permite acesso mesmo com bucket privado**

---

## 💡 Dica de prova
Se a questão falar em:
- "dados não estruturados em larga escala" → **Object Storage**
- "acesso temporário seguro via URL" → **PAR**
- "otimizar custo automaticamente" → **Auto-Tiering**
