# 💾 OCI Block Volume — Resumo

## 📌 O que é Block Volume?
O **Block Volume** é um armazenamento em bloco **persistente e durável** usado por instâncias de computação na OCI.

- Funciona como um **disco adicional (HD/SSD virtual)**
- Pode ser:
  - Criado
  - Anexado (attach)
  - Desanexado (detach)
  - Excluído
- 🔥 **Independente da instância**
  - Se a VM for deletada → o volume continua existindo

---

## ⚙️ Características principais

- 💾 **Persistente** → dados não se perdem ao excluir a VM
- 🛡️ **Durável** → múltiplas cópias dos dados
- 🔗 Pode ser anexado a instâncias conforme necessidade

---

## 🚀 Tipos de Block Volume

| Tipo                  | Uso |
|-----------------------|-----|
| Baixo custo           | Streaming / baixo desempenho |
| Equilibrado           | Uso geral (ex: boot) |
| Alto desempenho       | Aplicações exigentes |
| Ultra alto desempenho | Bancos de dados (alta performance) |

---

## 🤖 Auto-Tune Performance

- Ajusta automaticamente o desempenho do volume
- Quando:
  - 🔌 Desanexado → reduz custo
  - 🔗 Reanexado → volta ao desempenho anterior
- Ajuda na **otimização de custos**

---

## 🔐 Segurança

- 🔒 Criptografia:
  - Ativa por padrão (não pode desativar)
  - Em repouso + em trânsito
- 🔑 Pode usar:
  - Chaves da Oracle
  - Chaves próprias (Vault)

---

## 🧩 Recursos avançados

- 🔗 **Volume compartilhado**
  - Um volume pode ser usado por várias VMs

- 📏 **Redimensionamento**
  - Online (sem parar VM)
  - Offline (com detach)

- 🌍 **Replicação**
  - Assíncrona entre regiões
  - Usos:
    - Disaster Recovery
    - Migração
    - Expansão global

- 📦 **Volume Groups**
  - Agrupa vários volumes
  - Facilita backups

---

## 💽 Backup

---

## 🔌 Tipos de anexação

| Tipo             | Descrição |
|------------------|----------|
| iSCSI            | Gerenciado pelo sistema operacional |
| Paravirtualizado | Mais simples e comum (virtualizado) |

---

## 🖥️ Processo básico

### Criar volume
1. Storage → Block Volumes
2. Create Block Volume
3. Definir:
   - Nome
   - Tamanho
   - Performance
   - Auto-Tune (opcional)

### Anexar à VM
1. Attach to instance
2. Escolher tipo (Paravirtualizado recomendado)
3. Definir acesso (Read/Write)
